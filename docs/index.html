Session Abstract

Every plan set has keynote callouts, notes, and tables. Every revision means hours of manual editing — renumbering tags, reformatting MText, updating attributes, sheet by sheet, discipline by discipline. What if ONE Dynamo run could replace all of that at the click of a button?

This session demos a bi-directional engine that connects a single Excel workbook to every callout tag and table across every layout. Auto-filtered per sheet. Built-in orphan detection. No Data Links. No per-sheet tabs. From Standard Library to Civil 3D automation — an engine that works with ANY standard, ANY discipline, ANY scale. See it live. Configure it for your standards. Deploy it tomorrow.

Cell-by-Cell Editing
Every table row touched by hand. Every sheet. Every discipline. No batch operation exists in native Civil 3D.
Callout Tags Disconnected
MLeader attributes and table rows are independent text objects. Change one — the other doesn't know. Orphans and ghosts are the result.
Renumbering Cascades
One note renumbered means touching every callout on every sheet that references it. Across six disciplines. Manually.
No Standard Master List
Data Links require one Excel tab per table per sheet. A 50-sheet set means 50 separate links — each configured and maintained individually.
Standards Vary Everywhere
Jurisdiction to jurisdiction. Firm to firm. Reviewer to reviewer. Block names, table styles, note wording — nothing is standard.
QC Is Eyeball-Only
Print. Mark off. Hope you caught them all. No automated check exists to confirm every callout has a matching table entry.

Part 2 — The Solution: Keynote Suite v7.0

Production-deployed

  • Tables only — callouts untouched
  • No per-sheet auto-filtering
  • Renumber = touch every callout manually
  • No orphan/ghost detection

Keynote Suite v7.0

  • One master workbook drives everything

  • Renumber in Excel — sync handles the rest
  • Built-in orphan detection + diagnostic log
  • Linear scaling — same tool for 10 or 200 sheets

4 Operating Modes

Mode 1
Sync Export
CAD → Excel
Reads all keynote entities from the drawing. Exports to Excel with one sheet per layout. Preserves existing workbook — onboarding ramp for existing drawings.
Mode 2
Sync Import
Excel → CAD
Daily driver. Engineer updates Excel — run Sync Import — all callout descriptions update in-place by matching ID. Handles renumbering automatically.
Mode 3
Generate Table
Excel → CAD (new)
Creates keynote tables from Excel data. Places tables at configured location with correct styles, column widths, and callout blocks.
Mode 4
Export Table
CAD Table → Excel
Extracts existing table content to Excel for editing or review. QC receipt — confirm what is live in the drawing without modifying anything.

Supported Entity Types

📝

📦
